二、重点案例详解

1. Amazon Nova Pro 零训练视觉缺陷检测

来源链接：AWS - Implement Zero-Training Visual Defect Detection in Manufacturing with Amazon Nova Pro

应用场景：缺陷样本不足、产品变化频繁、传统视觉模型训练成本高的质检点。

核心做法：用参考图与待检图输入多模态大模型，通过结构化 prompt 要求模型输出缺陷类型、原因、置信判断和 bounding box；通过 API Gateway、Lambda、Bedrock 形成轻量服务。

使用技术：Amazon Nova Pro、Amazon Bedrock、API Gateway、Lambda、图像预处理、JSON 解析、prompt 工程。

光伏启发：包装外观、标签缺失、接线盒铭牌、托盘混料、工装摆放错误等低节拍场景，可以先用多模态模型做“人机复核”。

落地想法：建设“低样本视觉检查助手”，先接入离线图片或工位相机，对组件标签、条码、边框划伤、接线盒胶量外观、托盘摆放进行 OK/NOK 复核，并把结果写回 MES 异常单。

开发难度：中等。MVP 难点不在模型，而在现场拍照稳定性、输出 JSON 校验、误报复核界面、与 MES/质量单据对接。

预期收益：缩短新检测点上线周期；减少人工抽检漏项；为后续训练专用小模型积累标注数据。风险是单图推理速度不适合直接接到高速主线剔除。

2. BMW VSAS 视觉与声音分析服务

来源链接：AWS - BMW Group VSAS Case Study

应用场景：跨工厂图像/音频数据统一存储、检索、标注和 AI 分析。

核心做法：BMW 将原本分散的图像、音频和计算机视觉产品统一为 VSAS，支撑 16 个生产基地，处理平均每日 130 万个图像和音频文件，管理 1.3 PB 数据，并用 OpenSearch 做元数据查询。

使用技术：Amazon S3、S3 Intelligent-Tiering、OpenSearch、ECS、EC2、Graviton、Transfer Family、元数据索引、数据分层存储。

光伏启发：AI 质检规模化的基础不是单个模型，而是“图片和元数据能被找回来”。

落地想法：建立组件图像数据底座：EL、外观、层压后检、包装、IV 曲线截图、设备报警截图统一归档；用组件 SN、工单、设备号、班次、供应商批次、缺陷码做索引。

开发难度：中高。需要定义数据标准、采集接口、存储生命周期和权限；但可先从一个车间或一条线做轻量版本。

预期收益：减少质量追溯取证时间；提高 AI 模型训练样本召回效率；支持客户投诉时快速定位同批风险组件。

3. 凌云光光伏电池片 AI 智检

来源链接：凌云光 - 光伏电池片 AI 智检

应用场景：丝网印刷 AOI、隐裂检测、PL 检测、多工艺段质量追溯和换型模型迭代。

核心做法：AOI + F.Brain + GMQM，结合光学成像优化、深度学习缺陷分类、云边端质量平台，把生产数据和质量分析数据关联起来。

使用技术：AOI、PL 光致发光检测、近红外光源、深度学习分类、模板比对、在线样本采集、模型训练/封装/部署、质量追溯平台。

光伏启发：该案例最有价值的是“模型迭代链路”而不是单个检测模型。光伏产线换型频繁，模型从采样、训练到部署必须在线闭环。

落地想法：在组件 EL/AOI 工位增加“缺陷样本自动回流池”：复判员确认的误报、漏检、突发缺陷自动进入训练集；按产品型号、设备、班次建立版本化模型包。

开发难度：中高。若先做离线训练集管理，难度可降到中等。

预期收益：降低误判停线和复判人力；缩短新产品/新网版/新材料导入后的检测稳定周期。时效说明：来源发布于 2025-04-27，略早于最近一年窗口，但光伏产线可借鉴性高，保留。

4. Arm/Neuton.AI TinyML 预测性维护

来源链接：Arm - Predictive maintenance for smart factories

应用场景：泵、压缩机等旋转设备的异常检测与故障预警。

核心做法：在资源受限的微控制器上运行 TinyML 模型，实时处理振动、温度、压力等传感器数据，在故障前识别异常模式。

光伏启发：对层压机真空泵、空压机、冷却水泵、焊接机风机、传送电机等关键设备，先做边缘异常分数就有价值。

落地想法：选择 3 台真空泵或空压机，加装振动和温度采集，建立“正常运行基线 + 异常趋势告警 + 维修记录回填”的小闭环。

开发难度：中等。传感器安装、数据清洗和阈值策略比模型本身更关键。

预期收益：减少突发停线；降低过度保养；帮助设备工程师把经验固化为数据规则。

5. PV-YOLOv12n 光伏缺陷检测

来源链接：Scientific Reports - Optimized YOLO based model for photovoltaic defect detection in electroluminescence images

应用场景：EL 图像中的裂纹、星裂、黑芯、断栅、位移等缺陷检测，以及 RGB 光伏缺陷检测。

核心做法：在 YOLOv12n 基础上加入 A2C2f 等改进模块，提升细粒度缺陷定位能力，并保持轻量参数量。论文报告在 PVEL-AD 和 Roboflow 数据集上 mAP@0.5 约 0.91，参数量约 2.6M。

光伏启发：组件厂可把它作为轻量检测模型设计方向，尤其是小目标和弱对比缺陷；不建议直接照搬论文权重到现场。

落地想法：用现有 EL 图片先做离线 benchmark：YOLOv8n/YOLO11n/Anomalib PatchCore/PV-YOLO 思路对比，指标包含漏检率、误报率、单图耗时、复判节省时间。

开发难度：中等到高。取决于缺陷标注质量、图像一致性、是否要实时剔除。

预期收益：形成自有 EL 缺陷模型评测体系，减少供应商算法黑盒依赖。

6. Kiwa 与 Intertek 的光伏质量数据闭环启发

来源链接：Kiwa 2025 PV Module Manufacturing Quality Report、Intertek CEA - Factory to Field Inspection Decisions

应用场景：工厂审核、生产监督、出货检验、现场 EL 测试和质量问题回溯。

核心做法：Kiwa 强调基于工厂审核、生产监督、出货检验的数据总结制造质量风险；Intertek CEA 强调何时使用 EL 测试，以及打通现场与工厂检查数据的重要性。

光伏启发：把现场返图、EL 复测、出货批次、物料批次、设备参数拉到同一张风险地图。

落地想法：建设“客户现场问题反哺看板”：按客户、项目、组件 SN、工单、BOM 批次、EL 缺陷类型、工艺窗口展示风险聚类。

开发难度：中等。难点是客户数据回传格式不统一，以及现场缺陷定义和工厂缺陷码的映射。

预期收益：提高客户投诉响应速度；发现供应商批次、工艺窗口或设备漂移导致的系统性风险。

四、结合光伏工厂的应用点子

点子 1：EL/AOI 缺陷样本回流与模型迭代平台

痛点：EL/AOI 误报、漏检和换型后模型退化，复判员确认结果没有沉淀到训练集。

解决思路：把设备图片、模型结果、人工复判、缺陷码、工单和组件 SN 打通，自动生成“待训练样本池”。

数据：EL 原图、AOI 图片、缺陷框/分割 mask、复判结果、产品型号、设备号、班次、工艺参数。

技术方案：FiftyOne 管理样本；Anomalib/YOLO 做模型 baseline；PostgreSQL 存元数据；MinIO/S3 存图片；FastAPI 提供复判接口。

复杂度：中高。收益：缩短模型迭代周期，降低误判停线，建立自有缺陷数据资产。

点子 2：多模态大模型质检复核助手

痛点：低频缺陷、包装/标签/外观问题样本少，专用模型训练不经济。

解决思路：将参考图、待检图和结构化检查标准输入多模态模型，输出 OK/NOK、缺陷原因和坐标；用于抽检和复判，不直接控制高速剔除。

数据：工位图片、参考图、SOP 检查项、历史缺陷示例、MES 工单信息。

技术方案：本地视觉模型或云端多模态模型；JSON schema 校验；人工确认界面；结果回写 QMS/MES。

复杂度：中等。收益：一周内可做 MVP，适合新工位快速验证 AI 价值。

点子 3：层压机/真空泵边缘预测维护

痛点：真空泵、空压机、冷却水泵突发异常会造成停线或批量质量风险，传统点检依赖人工经验。

解决思路：用低成本振动、温度、电流传感器采集关键设备健康数据，建立正常基线和异常趋势告警。

数据：振动 RMS/频谱、温度、电流、压力、启停状态、维修记录、设备报警。

技术方案：边缘采集网关 + MQTT + 时序数据库 + 异常检测模型；初期用阈值和 Isolation Forest，后续试 TinyML。

复杂度：中等。收益：减少突发停线，辅助设备维护计划从定期转向按状态。

点子 4：客户现场问题到工厂批次的质量风险地图

痛点：客户现场 EL/热斑/功率衰减问题回传后，难以快速关联工厂工单、BOM 批次和设备参数。

解决思路：建立“现场缺陷码 - 工厂缺陷码 - 组件 SN - 工单 - BOM - 工艺窗口”映射，自动聚类同批风险。

数据：客户返图、现场 EL/红外图、组件 SN、出货批次、BOM 批次、EL 终检结果、IV 数据、层压参数。

技术方案：质量数据仓库 + 缺陷码字典 + 聚类分析 + 风险看板；先从重点客户项目开始。

复杂度：中高。收益：缩短客诉分析时间，提升召回/隔离决策准确性。

点子 5：能源与良率联动看板

痛点：能耗优化常只看总电量，不知道节能措施是否影响良率、返修率或节拍。

解决思路：把车间电表、空压、温湿度、设备状态和质量指标按工单/小时对齐，识别“高能耗低良率”的工艺段。

数据：电表、气耗、温湿度、设备 OEE、产量、良率、返修、报废、工艺参数。

技术方案：SCADA/MES 数据接入 + 时序库 + BI 看板 + 异常规则；后续做能耗预测。

复杂度：中等。收益：避免只降能耗不看质量，支持精益改善和碳足迹核算。

五、今日最值得尝试的一个小项目

EL/AOI 缺陷样本回流 MVP

推荐原因：今天的案例共同指向一个结论：光伏工厂做 AI 质检，最先要补的是数据闭环。相比直接追求高精度模型，先把图片、复判、缺陷码和工单关联起来，更容易产生持续收益。

MVP：上传或自动导入一批 EL/AOI 图片；绑定组件 SN、工单、设备号、产品型号；复判员标记 OK/NOK、缺陷类型、误报/漏检；系统生成按缺陷、型号、设备、班次筛选的数据集；导出 YOLO 或 Anomalib 可用的训练/评测目录。

所需数据：最近 1-2 周 EL/AOI 图片 1,000-5,000 张；MES 工单和组件 SN 映射；现有设备判定结果；人工复判结果，初期 200-500 张也可启动。

技术步骤：

  1. 用 MinIO 或本地文件夹存图，PostgreSQL/SQLite 存元数据。
  2. 用 FastAPI 建图片列表、筛选、复判、导出接口。
  3. 用简单 Web 页面做复判：图片、模型结果、缺陷码、备注。
  4. 接入 FiftyOne 或自建导出脚本，形成训练集版本。
  5. 选 1 个缺陷类别跑 Anomalib/YOLO baseline，输出误报漏报清单。

周期：3-5 天做离线导入 + 复判页面 + CSV/JSON 导出；1-2 周接入一台设备图片目录；3-4 周接入模型训练和评测报告。

扩展：增加多模态大模型复核；建立模型版本、训练集版本、上线记录；与 MES/QMS 异常单联动；做“缺陷热力图 + 批次风险地图”。
